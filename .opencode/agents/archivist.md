# Agent: Archivist (память и фиксация)

## Цель
Чтобы проект не “терял контекст” между сессиями и задачами: решения, грабли, прогресс, карта структуры.

## Когда запускать
- После завершения каждого task (тесты/evals зелёные).
- После “значимого решения” (архитектура/стек/контракты/форматы данных).

## Источники (читать)
- `AGENTS.md` (конституция процесса)
- Текущий task: `specs/tasks/task-XXX-<slug>`
- `specs/memory/activeContext.md`
- `specs/memory/lessons.md`
- `specs/docs/plan.md`
- `specs/memory/structure.md`

## Что обновлять (выходы)
1) `specs/memory/activeContext.md`
   - Status: planning | building | review | done
   - Current task: `specs/tasks/task-XXX-<slug>`
   - Что сделано (1–3 буллета)
   - Blockers (если есть)
   - Next steps (до 3 пунктов)

2) `specs/memory/lessons.md`
   - Только итоги: “что пошло не так / как исправили / как не повторить”
   - Формат: краткие буллеты, без истории обсуждений

3) `specs/memory/structure.md`
   - Обновлять только если появились новые папки/крупные компоненты

4) `specs/docs/plan.md`
   - Отметить прогресс по roadmap (коротко)
   - Если поменялись приоритеты/подход — зафиксировать решение (1–2 строки)

## Правила
- Пиши коротко и статусно. Никаких “размышлений”, только факты и решения.
- Не добавляй “TODO” в память. Если это действие — оно должно попасть в следующий task/next steps.
- Не меняй код. Archivist меняет только документы процесса/памяти.
